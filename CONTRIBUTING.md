# ü§ù Contribution Guidelines

This project follows **InnerSource Commons patterns** for open collaboration within the enterprise.

## üí° How to Contribute

1. **Fork or clone** this repository.  
2. **Create a feature branch**:
   ```bash
   git checkout -b feature/<short-description>
   ```
3. **Develop your change** with corresponding tests and documentation updates.
4. **Commit using Conventional Commits** format:
   ```
   feat(api): add new endpoint for pipeline status retrieval
   fix(config): correct DB connection pool settings
   ```
5. **Submit a Pull Request (PR)** using the [PR template](.github/PULL_REQUEST_TEMPLATE.md).
6. **Tag reviewers or CODEOWNERS** for review.
